# AIoT Innovation Lab

The Innovation Lab captures the creative systems thinking behind this AIoT
project. It organizes the repository into experiments, data flows, and
human-centered design patterns.

## Lab mission
Build AIoT solutions that are:
- empathetic: sensitive to people and environments,
- adaptive: able to learn and evolve over time,
- transparent: easy to understand and trust,
- practical: ready for real deployment.

## Lab zones
### 1. Scenario Studio
The project explores multiple AIoT scenes with different user needs:
- Smart mattress: health sensing with comfort-aware adjustments.
- Autonomous drone: safe delivery with swarm cooperation.
- Security gateway: continuous trust scoring and fraud detection.
- Smart grid: resilient power monitoring and anomaly alerting.
- Air quality guardian: proactive environmental risk warnings.

### 2. Data & AI architecture
Use this structured data flow as a design reference:
- Data capture: sensors extract meaning from temperature, motion, air and
  electrical signals.
- Edge preprocessing: noise reduction, sanity checks, and lightweight scoring.
- Local inference: classify events and trigger alerts in milliseconds.
- Cloud coordination: collect trends, update models, and share operational
  intelligence.

### 3. Creative experiment cards
Each card describes a project experiment:
- **Persona**: who benefits from the system.
- **Need**: what problem it solves.
- **Sensors**: what inputs it uses.
- **Intelligence**: how the system reasons.
- **Output**: what action it takes.

Example: Smart mattress
- Persona: restless sleeper.
- Need: improved sleep quality without intrusive monitoring.
- Sensors: pressure, temperature, motion.
- Intelligence: sleep phase detection and environment adjustment.
- Output: gentle heating, alert, data summary.

## Lab workflow
1. Define the scenario and persona.
2. Map sensors to edge and cloud processing.
3. Build a lightweight prototype in `code/aiot_demo.py`.
4. Document the idea in the `docs/` folder.
5. Deploy the prototype with Docker or Kubernetes.

## Innovation playbook
The lab encourages:
- modular design: reusable components for new AIoT scenarios.
- narrative thinking: story-driven requirements for user experience.
- technical creativity: merging edge intelligence with cloud learning.
- operational realism: deploy-ready artifacts and visual dashboards.

This repository creates the following outputs:
- `README.md`: project map, usage, and deployment guidance.
