# AIoT Dashboard Concept

This concept document describes a visual dashboard for the AIoT project.
The dashboard is designed to make the AIoT system tangible, accessible, and
insightful for operators, designers, and stakeholders.

## Dashboard Vision
The AIoT dashboard should act as a mission control center for the three
scenarios:
- Sleep Sanctuary
- Sky Courier
- Trust Guardian

It should present the state of the system using simple, high-impact visual
components and narrative explanations.

## Key Dashboard Sections
### 1. Live Status Panel
- Current scenario name and mode.
- Overall health status (OK / Warning / Alert).
- Active edge alerts and anomaly count.

### 2. Sensor Snapshot
- Real-time sensor values for each scenario.
- Visual gauges for critical metrics:
  - temperature and motion for Sleep Sanctuary
  - battery, obstacle distance, GPS accuracy for Sky Courier
  - trust score, audio energy, stability index for Trust Guardian

### 3. Insight Timeline
- A timeline chart showing events and anomalies over time.
- Highlights when edge inference detects a change in status.
- Enables quick review of system behavior and model drift.

### 4. Action & Response Panel
- Suggested actions generated by AIoT, such as:
  - adjust sleep environment,
  - reroute a drone,
  - raise a trust warning.
- Confidence and explanation text for each suggestion.

### 5. Design Notes and Storyboard
- A compact story card that explains the scenario context.
- Notes on user impact, privacy considerations, and next improvements.

## Visual Style Recommendations
- Use simple, high-contrast cards and gauges.
- Keep the layout modular so panels can be rearranged.
- Favor clear iconography for alerts, health, and context.
- Include a muted dark mode for operations-focused display.

## Dashboard Data Flow
1. Edge devices stream telemetry in lightweight JSON format.
2. The dashboard backend aggregates data and applies baseline logic.
3. Visual panels render current state, recent events, and recommended actions.

## Example Panel Layout
- Top row: Live Status Panel, Scenario Overview
- Middle row: Sensor Snapshot, Insight Timeline
- Bottom row: Action & Response Panel, Storyboard Notes

## Why this matters
A dashboard makes AIoT systems easier to understand and trust.
It transforms raw sensor data and AI inference into an operational narrative
that can be used by engineers, domain experts, and decision makers.
